# CodingDatafy

**On a mission to build the world's largest reference and knowledge base for coding languages.**

---

### About CodingDatafy
**CodingDatafy** is an open-source organization dedicated to mapping the programming landscape with ultimate precision and structural clarity. We adopt an **AI-First, highly-structured data approach** to technical writing, ensuring that developers and LLMs worldwide have access to a comprehensive, version-controlled knowledge base.

### Architecture & Tech Stack
Our ecosystem is engineered as a decoupled, database-less architecture designed for sub-millisecond edge delivery, maximum scalability, and global performance:

* **Core Engine:** [Next.js 16](https://nextjs.org/) backed by **TypeScript** (Static Site Generation via Edge Nodes).
* **Infrastructure:** Database-less, flat-file architecture utilizing structured `MD` and `JSON` schemas.
* **Edge & Deployment:** Hosted on [Vercel](https://vercel.com/) working seamlessly alongside [Cloudflare](https://www.cloudflare.com/) for global DNS routing and advanced caching.

### Ecosystem Repositories
To prevent vendor lock-in and optimize build consumption, our architecture is split into two specialized hubs:
* **[`centroidium`](https://github.com/codingdatafy/centroidium):** The high-performance, decoupled open-source website engine.
* **[`content`](https://github.com/codingdatafy/content):** The pure data repository containing all site files and schemas inside the `data/` directory.

---

### Global Contributions
We operate with the strict discipline of a global engineering team. Every commit is linked to a specific issue, and every issue tracks back to an active project. 

* **Branch Strategy:** We protect production integrity using a rigorous `develop` to `main` automated testing workflow.
* **Become a Contributor:** Read our comprehensive workflow rules on the [Contribute Page](https://www.codingdatafy.com/contribute).
* **Report or Suggest:** Found a syntax error or want to map a new language? Open a [Formal Issue](https://github.com/codingdatafy/content/issues).

### Licensing
To guarantee open access to information while guarding core engineering efforts, we apply a strict dual-licensing policy:
* **Software & Tooling (`centroidium`):** Licensed under the permissive [MIT License](https://github.com/codingdatafy/.github/blob/main/LICENSE).
* **Technical Content (`content`):** All documentation and structural data files follow the [CC BY-SA 4.0](https://creativecommons.org/licenses/by-sa/4.0/) international license.
